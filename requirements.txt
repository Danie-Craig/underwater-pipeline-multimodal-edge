#  This is the CLOUD-GPU / server side: train + fine-tune both YOLO11 models,
#  run the augmentation + robustness pipeline, and export to ONNX.
#
#  Install (Linux, NVIDIA GPU, Python 3.10+), inside a venv:
#      python -m venv .venv && source .venv/bin/activate
#      pip install --upgrade pip
#      pip install -r requirements.txt
#      python scripts/verify_setup.py        # confirms CUDA + all imports
#
#  Versions are pinned to lower bounds for portability across CUDA/driver
#  combos. After your first clean install, freeze exact versions for
#  reproducibility:   pip freeze > requirements.lock.txt
#
#  NOTE on PyTorch: `pip install torch` ships CUDA wheels by default on Linux.
#  If your CUDA/driver needs a specific build, follow https://pytorch.org
#  and install torch/torchvision FIRST, then the rest of this file.
#
#  NOTE on ONNX Runtime: install onnxruntime-GPU (CUDA execution provider) for
#  the server-side benchmark. Do NOT also install plain `onnxruntime` — having
#  both in one env causes provider conflicts. (The Jetson uses a different,
#  Jetson-specific wheel — see requirements-jetson.txt.)
